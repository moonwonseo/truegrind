# TrueGrind Deployment Handoff
*Date: April 13, 2026*

This document covers everything done to get TrueGrind deployed with a live backend (Modal) and live frontend (Vercel), plus known issues discovered during testing.

| Layer | Platform | URL |
|---|---|---|
| Backend (FastAPI + YOLOv8) | Modal | `https://moonwonseo--truegrind-api-fastapi-app.modal.run` |
| Frontend (SvelteKit) | Vercel | `https://truegrind.vercel.app` |

## What Was Done

### 1. Cloudflare R2 Setup
- Created R2 bucket named `truegrind`
- Created R2 API token named `truegrind-r2` with Object Read & Write permissions
- Uploaded `best.pt` to R2 at path `models/models/best.pt` using `r2_sync.py`
- Credentials stored in local `.env` file (gitignored)

### 2. Modal Backend Deployment
- Installed Modal CLI: `pip3 install modal`
- Authenticated: `python3 -m modal setup`
- Created Modal secrets: `python3 -m modal secret create truegrind-secrets` with:
  - `R2_ACCESS_KEY_ID`
  - `R2_SECRET_ACCESS_KEY`
  - `R2_ACCOUNT_ID=5c88cde63e8dcbf2099cd599bd0d56fd`
  - `R2_BUCKET_NAME=truegrind`
  - `R2_MODEL_PREFIX=models/models`
  - `MODEL_PATH=best.pt`
- Created `modal_app.py` in project root — wraps `api_server.py` as a Modal ASGI app
- Key Modal config decisions:
  - `gpu="T4"` — uses NVIDIA T4 GPU for inference
  - `numpy<2` pinned to fix NumPy 1.x/2.x compatibility issue with torch
  - `.apt_install("libgl1", "libglib2.0-0")` required for OpenCV on Debian
  - `.add_local_dir(".", remote_path="/root")` to copy project files into container
  - Model cached in a Modal Volume (`truegrind-model-cache`) so it doesn't re-download on every cold start
  - `scaledown_window=300` — container stays warm 5 min after last request

- Deploy command: `python3 -m modal deploy modal_app.py`

### 3. CORS Fix
- `api_server.py` originally had `allow_origins=["http://localhost:5173"]`
- Changed to `allow_origins=["*"]` to allow Vercel frontend to call Modal backend
- For production, tighten to: `allow_origins=["http://localhost:5173", "https://truegrind.vercel.app"]`

### 4. Vercel Frontend Deployment
- Connected `moonwonseo/truegrind` GitHub repo to Vercel
- Set Application Preset: **SvelteKit**
- Set Root Directory: **frontend**
- Added environment variable:
  - `VITE_API_BASE_URL = https://moonwonseo--truegrind-api-fastapi-app.modal.run`
- Frontend auto-redeploys on every `git push` to `main`

### 5. API Base URL — Environment Variable Setup
- `frontend/src/lib/api.ts` updated to use env var instead of hardcoded localhost:
  ```typescript
  const API_BASE = (import.meta.env.VITE_API_BASE_URL ?? 'http://localhost:8000') + '/api';
  ```
- `frontend/.env.local` created locally with `VITE_API_BASE_URL=http://localhost:8000`
- This means:
  - **Local dev (Antigravity):** calls `localhost:8000`
  - **Vercel production:** calls Modal URL

## Known Issues

### HEIC vs PNG giving different readings
- **What happens:** uploading the same photo from desktop (PNG) vs iPhone (HEIC) gives different D50 readings
- **Why:** HEIC is Apple's compressed format. When uploaded through the browser, it gets converted on the fly at varying quality/resolution, which affects quarter detection accuracy and particle sizing
- **Current workaround:** Tell users to set iPhone to JPEG mode: **Settings → Camera → Formats → Most Compatible**
- **Proper fix (not yet implemented):** Server-side HEIC→JPEG conversion in `api_server.py` before the image hits the pipeline. Requires `heif-convert` system dependency which adds complexity to the Modal image build.

### Modal cold start latency
- When the container hasn't been used for 5+ minutes it spins down
- First request after idle takes ~10-15 seconds to cold start
- **Workaround for demos:** hit `/api/health` first to warm it up before showing the demo
- **Proper fix:** increase `scaledown_window` or use Modal's keep-warm feature (costs more compute credits)

### `models/models/best.pt` double-nested path
- Due to an upload issue, `best.pt` ended up at `models/models/best.pt` in R2 instead of `models/best.pt`
- `R2_MODEL_PREFIX=models/models` is set in Modal secrets to compensate
- Not a problem but worth cleaning up eventually
